## 一、工具
### 1.Edge 浏览器
**解决的问题**  在本地预览与调试前端应用，验证布局、响应式与交互是否符合预期；查看vue、Tailwind CSS官网，以其中部分内容作为开发参考
**使用场景**    运行 `npm run dev` 后，在Edge中打开Vite提供的本地地址进行功能测试、控制台调试、检查网络请求与LocalStorage
**关键步骤**    启动开发服务器；
               在Edge输入URL；
               使用开发者工具（F12）查看；
               调整视口宽度验证响应式布局与断点行为
### 2. Cursor
**解决的问题**  AI 辅助编写与重构
**使用场景**    编写/修改 Vue 组件、TypeScript 类型、Tailwind 样式
**关键步骤**    通过对话与内联编辑请求等方式协助实现需求与修复代码问题
### 3. UGit
**解决的问题**  对项目进行版本控制与协作：提交、分支、查看历史、解决冲突，便于回溯与交付
**使用场景**    完成需求后，用 UGit 做提交；
               在需要时创建分支或与远程仓库同步

## 二、与项目目标的对应关系
| 项目目标（README / Challenge） | 主要涉及工具 |
|--------------------------------|--------------|
| 实现搜索与空状态等 UI 行为 | Vue 3、Tailwind、Edge（验证） |
| 修复分类筛选不更新列表的 Bug | Vue 3（computed 响应式）、Cursor（编辑与 AI 辅助） |
| 改进 LocalStorage 持久化与安全序列化 | Vue 3（watch）、TypeScript、useLocalStorage |
| 补全 TypeScript 类型并保证类型安全 | TypeScript、ESLint（@typescript-eslint） |
| 保持布局响应式与无障碍 | Tailwind、Vue、Edge（视口与开发者工具测试） |
| 版本管理与协作 | UGit、Git |
